# API Testing Portfolio

## Overview

This section showcases my hands-on API testing practice using Postman.

The objective was to understand REST API functionality, validate responses, verify status codes, test CRUD operations, and implement automated assertions.

## Tool Used

* Postman

## API Tested

JSONPlaceholder REST API

Base URL:

https://jsonplaceholder.typicode.com/

## HTTP Methods Tested

### GET

* Retrieve all users
* Retrieve a single user
* Validate response data

### POST

* Create a new user
* Validate status code and response body

### PUT

* Update an existing user
* Verify updated data is returned

### PATCH

* Partially update user information
* Validate modified fields

### DELETE

* Delete a user record
* Verify API response

## Assertions Implemented

* Status Code Validation
* Response Time Validation
* Content-Type Validation
* Response Body Validation
* User Count Validation

## Key Learning Outcomes

* Understanding REST APIs
* Working with JSON data
* Using Environment Variables
* Creating Postman Assertions
* Performing Positive and Negative API Testing
* Validating API Responses
